# M1-W02 — FastAPI-обёртка над LLM (gateway)

> Статус: 🔲 не начат · Приоритет: P0 · Трек: FastAPI/LLM · Оценка времени: 10–14 ч

## Цель недели
Превратить разовый скрипт W01 в сервис: FastAPI с эндпоинтами `/ask` и `/health`, валидацией Pydantic, базовым логированием и тестами на pytest. Это фундамент всех будущих проектов.

## Что понять (→ `theory.md`)
- Зачем LLM-вызов прятать за свой API (а не дёргать провайдера из фронта).
- Request/response модели Pydantic как контракт.
- `/health` и почему он нужен для Docker/прода.
- Базовая обработка ошибок и таймаутов.

## Что сделать (→ `practice.md`)
1. `POST /ask` — принимает вопрос, возвращает ответ модели.
2. `GET /health` — статус сервиса.
3. Pydantic-модели запроса/ответа.
4. ≥3 теста на pytest (включая мок LLM, без реального вызова).

## Гейт недели (зачёт)
- [ ] `/ask` возвращает валидный по схеме ответ;
- [ ] `/health` отвечает 200;
- [ ] ≥3 pytest зелёные, LLM замокан (тесты не ходят в сеть);
- [ ] секреты из `.env`, не в коде;
- [ ] `assessments/weekly/W02.md` заполнен.
