# STEPIC_ROADMAP — порядок прохождения курсов Stepik

Дата фиксации: 2026-06-03.

Цель: использовать выбранные курсы Stepik как внешний учебный трек к репозиторию `ai_sa_ba`, но не заменять основной курс в `COURSE.md`. Каждый курс должен давать артефакт в GitHub: конспект, практическую работу, issue и pull request.

## Логика порядка

1. Сначала инструменты разработки: Git/GitHub и Docker.
2. Затем Python как обязательная база для ML, LLM и агентов.
3. Затем классическое машинное обучение: сначала обзорно-аналитический курс, потом более технический.
4. Затем нейронные сети.
5. Затем LLM.
6. Последним — разработка ИИ-агентов, потому что там нужны Python, API, базовое понимание LLM, Docker и Git.

## Почему именно так

### Git/GitHub — до всего остального

Учебный процесс в этом репозитории построен вокруг артефактов: issue, branch, commit, pull request, review. Поэтому Git/GitHub закрывается первым, до Python и Docker.

### Python — до ML, нейросетей и агентов

Без Python дальнейшие курсы будут превращаться в просмотр теории без самостоятельной практики. Python нужен для задач, ноутбуков, API, прототипов, RAG и агентов.

### Docker — рано, но после базового Git

Docker нужен не для теории ML, а для воспроизводимости практических работ: запуск сервисов, API, векторной БД, бота, агента. Поэтому Docker ставится до крупных проектов, но после Git.

### Два курса по ML не дублируют друг друга полностью

Курс `Машинное обучение` лучше пройти первым как обзорный и аналитический: он помогает говорить с заказчиком, понимать постановку задач, ограничения и применимость ML.

Курс `Добрый, добрый ИИ. Часть 1. Машинное обучение` ставится после него как основной технический курс, где нужно больше математики, Python и практической реализации алгоритмов.

### Нейронные сети — после классического ML

Нейронные сети проще понимать после линейных моделей, градиентных методов, переобучения, регуляризации, метрик и базовой оптимизации.

### LLM — после нейросетей

LLM — частный, более сложный класс нейросетевых моделей. Перед курсом по LLM нужно понимать хотя бы базово: что такое модель, обучение, параметры, функция потерь, обобщение, переобучение.

### ИИ-агенты — последними

Агент — это не только LLM. Это LLM + инструменты + память + состояние + планирование + внешние API + обработка ошибок + безопасность + наблюдаемость. Поэтому курс по агентам должен идти после Git, Python, Docker и базового понимания LLM.

## Как проходить каждый курс

Для каждого курса заводить issue по шаблону:

```md
# Stepik: <название курса>

## Цель
- Что должен понять/уметь после курса.

## Артефакты
- [ ] Конспект в `docs/`
- [ ] Практика в `projects/` или `modules/`
- [ ] 5–10 выводов для системного аналитика
- [ ] Ограничения технологии / когда не применять
- [ ] Pull request с результатом

## Критерий зачёта
Курс засчитан только если есть PR с артефактом, а не только просмотр видео.
```

## Рекомендуемый темп

| Этап | Курсы | Оценка по времени |
|---|---|---:|
| Подготовка | Git/GitHub, Python, Docker | 3–5 недель |
| ML-база | `Машинное обучение`, `Добрый, добрый ИИ. Часть 1` | 4–7 недель |
| Deep Learning / LLM | `Нейронные сети`, `Внутри LLM` | 3–5 недель |
| Agents | `Введение в разработку ИИ-агентов` | 2–4 недели |

Итого: 12–21 неделя при нормальном темпе и наличии практики. Если делать только просмотр без артефактов — прогресс не засчитывать.

## Минимальный маршрут, если времени мало

Если нужно сократить маршрут, оставить:

1. Основы Git и GitHub.
2. "Поколение Python": курс для начинающих.
3. Docker для начинающих + практический опыт.
4. Машинное обучение.
5. Внутри LLM: как думает ChatGPT?
